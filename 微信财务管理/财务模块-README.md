# 财务模块 README

一个本地优先的微信钱包账单可视化原型。目标是读取微信官方导出的个人对账 CSV，在浏览器本地完成清洗、统计和图表展示，不上传原始账单。

## 调研结论

- 成熟开源案例主要走“导出账单 CSV → 清洗字段 → 分类统计 → Excel / Markdown / 图表展示”的路线。
- `MickLife/KeepAccounts_v2.0` 偏 Excel 自动记账：合并微信、支付宝账单，补分类，并用 Excel 透视表做可视化。
- `zxc7563598/alipay-wechat-finance` 偏 Python 报告：把微信、支付宝 CSV 统一字段，生成 Markdown 财务报告。
- 微信个人账单通常从微信 App 的“我 → 服务 → 钱包 → 账单 → 常见问题 → 下载账单 → 用于个人对账”导出，解压后得到 `.csv` 文件。

## 当前版本

- 支持导入微信支付 CSV 文件。
- 内置浏览器本地账本库，每次导入后会保存历史交易。
- 按交易单号去重；如果账单缺少交易单号，会用交易时间、收支、金额、对象、商品和支付方式生成指纹去重。
- 自动识别微信账单常见字段：交易时间、交易类型、交易对方、商品、收/支、金额、支付方式、交易状态、备注。
- 支持本地解析 GB18030 / UTF-8 编码。
- 提供核心指标、月度收支趋势、分类占比、余额走势和交易明细。
- 提供搜索、月份筛选、收支筛选和交易类型筛选。

## 使用方式

直接打开 `index.html`，选择微信导出的 CSV 文件即可。也可以点击页面里的“载入示例”先看效果。

推荐节奏：

1. 每月或每季度从微信导出一次“用于个人对账”的 CSV。
2. 在页面中导入 CSV。
3. 如果导入时间范围有重叠，页面会自动跳过重复交易，只新增没见过的交易。
4. 历史账本保存在当前浏览器的 `localStorage` 中，下次打开页面会自动读取。

如果想重来，可以点击“清空本地账本”。这个操作只清浏览器里的账本，不影响原始 CSV 文件。

## 半自动导入预留

`imports/` 目录是后续 Mac 监听文件夹的入口。下一步可以做一个小脚本，自动扫描你丢进去的微信账单 CSV / ZIP，解压、解析、去重后写入本地账本文件。

## 隐私边界

当前原型不连接服务器、不写入云端，账单文件只在本机浏览器内解析。后续如果要接入主任务 App，建议仍然保留“只导入聚合统计，不保存原始交易明细”的模式。

## 后续方向
