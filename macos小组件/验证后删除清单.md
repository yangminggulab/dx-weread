为了验证 `macos小组件` 的 SwiftUI + WidgetKit 工程，临时安装以下工具/应用。验证完成后如果不继续做 macOS 原生开发，可以删除以释放空间。

## 当前状态

- 2026-05-29：准备安装 **Xcode 16.4**，用于验证 `macos小组件` 工程。
- 选择原因：本机是 macOS 15.7.3；Mac App Store 当前给的 Xcode 26.5 要求 macOS 26.2+，不兼容。
- 2026-05-29：已安装 `mas` 和 `xcodes`。`xcodes install 16.4` 需要单独输入 Apple ID/password，不能复用浏览器登录态。
- 当前继续方式：从 Apple Developer Downloads 手动下载 `Xcode 16.4 (16F6).xip`，然后移动到本目录或提供完整文件路径，再继续解压和安装。
- 2026-05-29：`Xcode.app` 已移动到 `/Applications/Xcode.app`，版本验证为 Xcode 16.4 (16F6)。
- `~/Downloads/Xcode_16.4.xip` 仍保留。工程验证完成后可以删除。
- 2026-05-29：`TaskWidgetApp` Debug 构建通过，产物位于 `.DerivedData/Build/Products/Debug/TaskWidget.app`，Widget 扩展产物位于 `.DerivedData/Build/Products/Debug/TaskWidgetExtension.appex`。
- 安装位置按 macOS 默认规则使用 `/Applications/Xcode.app`，不把 Xcode 本体放进项目目录。
- 本文件作为后续清理提醒：验证完成后删除 Xcode、XcodeGen、相关缓存。

## 需要删除

- `/Applications/Xcode.app`
  - 用途：构建和验证 WidgetKit / App Extension。
  - 预计占用：约 15-25 GB 或更多。
  - 删除方式：从 `/Applications` 移到废纸篓，或执行 `sudo rm -rf /Applications/Xcode.app`。

- Homebrew 包 `mas`
  - 用途：通过 Mac App Store 命令行安装 Xcode。
  - 预计占用：很小。
  - 删除方式：`brew uninstall mas`。

- Homebrew 包 `xcodegen`
  - 用途：由 `project.yml` 生成 Xcode 工程。
  - 预计占用：较小。
  - 删除方式：`brew uninstall xcodegen`。

- Homebrew 包 `xcodes`
  - 用途：下载兼容本机系统的旧版 Xcode 16.4。
  - 预计占用：较小。
  - 删除方式：`brew uninstall xcodes`。

## 可按需清理

- `~/Library/Developer/Xcode/DerivedData`
- `~/Library/Caches/com.apple.dt.Xcode`
- `~/Library/Developer/CoreSimulator`

这些目录是 Xcode 构建缓存和模拟器数据。只验证 macOS Widget 时，后续可以清理。
