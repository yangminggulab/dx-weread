# 数据同步架构说明

## 三端概览

| 端 | 入口 | 数据存储 |
|---|---|---|
| 小程序 | 直接调用 Cloudflare Worker | 云端 KV（`app_data` / `diary_data`） |
| 网页版·云端模式 | `yangminggu.com/tasks`（Worker） | 云端 KV |
| 网页版·本地模式 | `localhost:8080`（server.py） | 本地 `data.json` / `diary.json` |

---

## 数据流详解

### 任务（Tasks）

```
小程序
  └─ POST /api/tasks/add|update|delete → 云端 KV ←┐
                                                    │ 每15分钟拉取合并
网页云端版                                           │ + 推送回云端
  └─ POST /api/data → 云端 KV                       │
                                                    │
网页本地版                                           │
  └─ POST /api/data → 本地 data.json ──推送──────────┘
```

### 日记（Diary）

```
小程序
  └─ POST /api/diary → 云端 KV ←┐
                                 │ 每15分钟拉取
网页云端版                        │ + 推送回云端
  └─ POST /api/diary → 云端 KV   │
                                 │
网页本地版                        │
  └─ POST /api/diary → 本地 diary.json ──推送──────┘
```

### 本地 server.py 主动推云端的时机

| 触发条件 | 推送内容 |
|---|---|
| 本地网页版保存任务/笔记 | 任务+书单（`/api/data`） |
| 本地网页版保存日记 | 日记 |
| 微信读书同步完成 | 任务+书单 |
| 凌晨5点每日重置 | 任务+书单 + 日记 |
| 手动调用 `/api/diary/push` | 日记 |
| 手动调用 `/api/sync/pull` | 从云端拉取（反向） |

## 冲突优先级

### 任务

**云端为绝对权威**：
- 云端有、本地也有 → 比较 `updatedAt` / `createdAt`，**时间戳更新的一方获胜**；两者都没有时间戳则云端获胜
- 云端有、本地没有 → 加入本地
- 云端没有、本地有 → **删除本地**（云端每日重置删掉的任务不会复活）

### 日记

**以内容更长的一方为准**（避免覆盖掉更完整的版本）：
- 今日内容：本地与云端各清洗一遍，**字数多的获胜**
- 归档条目：按日期去重，同一天**字数多的获胜**
- 归档后内容为空的条目会被丢弃

### 书单 / 笔记 / 动态（books / notes / updates）

**本地为基础，云端补充**：
- 本地有、云端没有 → 保留本地
- 云端有、本地没有 → 加入本地
- 两端都有同一 ID → **时间戳更新的一方获胜**；无时间戳则本地获胜

---

## 归档顺序

### 日记归档

- 触发条件：今日日记的 `date` 字段与当前日期不符（跨天了）
- 插入位置：按 `date` 字符串升序排列（最早的在前），归档列表**永不删除**
- 归档后：`today` 重置为当天空内容
- 凌晨5点分界线：本地以5点为一天的分界（5点前仍算昨天），云端同

### 每日重置顺序（本地）

1. 从云端拉取最新数据并合并（保证本地是最新的）
2. 归档日记
3. 推送日记到云端
4. 清除已完成任务
5. 推送任务到云端

---

## 每日重置

两处独立运行：

- **云端**：Cron `0 21 * * *` UTC = 北京时间每天凌晨5点
- **本地**：`server.py` 内线程，每天5点（本地时区）

两处都执行：
1. 把今日日记归档（如果日期变了）
2. 清除已完成任务

## 已知逻辑问题

### ⚠️ 问题：日记归档时间依赖本地时区

本地 `server.py` 的 `effective_diary_date()` 使用 `datetime.now()`（无时区），如果 Mac 时区不是 CST（UTC+8）会导致归档日期与云端不一致。目前 Mac 在 CST，暂无影响。
