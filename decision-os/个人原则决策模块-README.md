# 个人原则决策模块 README

> 核心痛点：过去遇到的问题当时解决了、写进了日记，但一年后遇到类似问题时忘了解法。
> 这个模块的唯一目的：**让过去的自己能在未来帮到自己**。

---

## 这是什么，不是什么

**是：** 一本按问题类型分好类的个人解法手册，每个小类直链到对应日记条目。

**不是：**
- 不是日记的搜索功能
- 不是任务管理
- 不是通用知识库

**核心使用流程：**
```
现在遇到问题
  → 打开这个模块
  → 找到对应大类 → 对应小类
  → 点击直链到那篇日记
  → 读自己当时是怎么处理的
```

## 我的最新思想（2026 年）

这些是日记中最成熟、最近形成的思考：

**关于求真务实（2026-04-20）**
> 缺的不是宏大的道理，而是愿意认真看细节的人。遇到事情先不急着下结论：这件事真的是我以为的那样吗？差别在哪里？我能做什么？

**关于学习的非线性（2026-05-23）**
> 学习不是今天学 10% 明天学 10%，而是开始只学会了 1%，突然有一天想明白了学会了 50%，再有一天变成 80%。不要用线性进度来衡量自己。

**关于困难但正确的选择（2026-04-26）**
> 每一次回望，那些当时看起来最难的选择，事后都是对的。这条规律足够稳定，可以当原则用。

**关于自信的定义（2026-01-17）**
> 真正的自信不是觉得自己一次就能成功，而是失败千万次我都终将成功。

**关于苦难（2026-05-22）**
> 快乐久了原本值得高兴的时候也没有那么高兴，痛苦久了没那么值得快乐的事情也感觉快乐。痛苦就当刷新快乐阈值。

**关于不切实际的幻想（2026-05-29）**
> 很多苦恼是自己搞自己心态。写下来那个想法，再审查一遍：它真的成立吗？

**关于长期主义**
> 虽然底层不能有太高的品德约束，但长期主义的信念必须坚定。

## 设计方向

### 当前确定的原则

- **新模块，不改现有日记页**
- **三层结构**：大类 → 小类 → 日记直链
- **点日记链接**：打开当天完整日记，直接看原文
- **内容来源**：从 388 篇日记（2024-08 至 2026-05）提炼，不是手写的

### 已对比的设计方案

| 方案 | 描述 | 优缺点 |
|------|------|------|
| A. 网格卡片 | 大类=卡片，点击进子页面，子页面显示小类+链接 | 清晰两步到位，但需要点击翻页 |
| B. 三栏列视图 | 大类列 \| 小类列 \| 日记链接列，同时可见 | 最快，一屏三级，productivity 风格 |
| C. 单页手风琴 | 全部在一页，展开/折叠，不跳页 | 简单不迷路，文字密集时稍乱 |
| D. Bento 行内展开 | 不等大卡片，点击就地展开 | 最好看，开发量最大 |

### 待研究问题（下次上网搜）

**核心问题**：如何用**最小的层级**展示**尽可能多的东西**，且保持**最清晰**？

具体要搜的方向：
- Progressive disclosure 最佳实践：什么时候该隐藏，什么时候该暴露？
- Information density vs. cognitive load：密度和认知负担的平衡点
- 参考产品：Linear（三栏）、Apple 健康（类别+子类+图表一页）、Stripe Docs（左树+右内容）
- 一屏内同时呈现"分类导航"+"内容预览"+"直链"的设计案例
- 手风琴 vs. 三栏 vs. 侧滑，在中等信息量（7 大类 × 4 小类 × 5 条链接）下哪种最高效
- 是否有"只看结构"和"看全部内容"两种视图切换的设计先例
